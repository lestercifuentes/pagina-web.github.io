REDES DE COMPUTADORAS



¿QUE ES UNA RED DE COMPUTADORAS?

Una red de computadoras es, en esencia, un conjunto interconectado de dispositivos informáticos, como ordenadores, servidores, impresoras y dispositivos móviles, que pueden comunicarse entre sí y compartir recursos. Esta comunicación se basa en un conjunto de reglas y protocolos, conocidos como protocolos de red, que definen cómo los dispositivos deben interactuar para garantizar una transmisión de datos fiable y eficiente.

La capacidad de compartir recursos, como archivos, impresoras y conexiones a Internet, es una de las principales ventajas de las redes de computadoras. Además, las redes permiten la colaboración en tiempo real, facilitando el trabajo en equipo y el acceso a información centralizada. La arquitectura de una red puede variar significativamente, dependiendo de su tamaño, propósito y la topología que adopte.

TIPO DE REDES DE COMPUTADORAS

Una red de computadoras es un conjunto de dispositivos interconectados que comparten información, recursos o servicios. Su clasificación más común se basa en el alcance geográfico y la cobertura.

1. LAN (Local Area Network)

Conecta dispositivos en un área reducida como una casa, oficina o escuela.
Ejemplo: Una red WiFi doméstica.
Ventajas: Una alta velocidad, bajo costo.
Desventajas: Un alcance limitado.


2. MAN (Metropolitan Area Network)

Cubre áreas más amplias como ciudades o campus universitarios.
Ejemplo: Una red que conecta sedes de una universidad en la misma ciudad.
Ventajas: Una buena velocidad y cobertura media.
Desventajas: El mayor costo que una LAN, requiere infraestructura profesional.


3. WAN (Wide Area Network)

Interconecta redes a nivel nacional o internacional.
Ejemplo: El internet.
Ventajas: La cobertura global.
Desventajas: Los costos altos y mayor riesgo de vulnerabilidad.



Dispositivos de Red

Los dispositivos de red son componentes esenciales para la comunicación y gestión de datos en una red. Aquí algunos tipos comunes:

  • Routers: Conectan redes locales (LAN) con redes más amplias (WAN) y gestionan el tráfico de datos.
  • Switches: Unen dispositivos dentro de una misma red local, optimizando el flujo de datos.
  • Módems: Permiten la conexión de dispositivos a Internet y a otras redes.
  • Firewalls: Protegen la red de amenazas externas y filtran el tráfico no autorizado.
  • Access Points: Proporcionan conectividad inalámbrica a dispositivos en una red.

¿QUE ES UNA DIRECCION IP?


Una dirección IP es el identificador que permite que un dispositivo se comunique dentro de una red. Existen dos tipos principales: las **IP privadas** y las **IP públicas**. La IP privada es asignada por el router a cada dispositivo dentro de una red local, como computadoras, teléfonos o tabletas, y solo funciona dentro de esa red; por ejemplo, la dirección 192.168.0.1 es muy común para acceder a la configuración del router. En cambio, la IP pública es asignada por el proveedor de internet y es única en toda la red global, lo que permite que tu dispositivo sea reconocido en internet. Las direcciones IP siguen estándares definidos por la IANA: **IPv4**, que utiliza cuatro números separados por puntos y tiene un límite de 4 mil millones de direcciones, y **IPv6**, que usa ocho grupos de números hexadecimales y ofrece una cantidad prácticamente ilimitada de direcciones para cubrir la creciente demanda. Además, la IANA ha reservado rangos específicos para las IP privadas, como 10.0.0.0 – 10.255.255.255, 172.16.0.0 – 172.31.255.255 y 192.168.0.0 – 192.168.255.255. Esto significa que la misma dirección privada puede repetirse en diferentes redes sin problema.

Protocolos de Redes de Computadoras

Los protocolos de red son un conjunto de reglas y normas que permiten la comunicación efectiva entre dispositivos en un sistema informático. Estos protocolos definen cómo se transmiten, estructuran y procesan los datos en una red, asegurando que el origen y el destino puedan entenderse mutuamente.

    Tipos de Protocolos y Funciones


    Los protocolos se organizan según el modelo OSI, que consta de 7 capas. Cada capa tiene funciones específicas y protocolos asociados:

  • Capa de Aplicación: Interactúa directamente con el usuario. Ejemplos: HTTP, FTP, SMTP, DNS.
  • Capa de Presentación: Traduce los datos al formato adecuado. Ejemplos: AFP, NFS.
  • Capa de Sesión: Maneja la comunicación entre dispositivos. Ejemplos: NetBIOS, ISNS.
  • Capa de Transporte: Segmenta los datos y asegura su entrega. Ejemplos: TCP (fiable), UDP (rápido).
  • Capa de Red: Determina la mejor ruta para los datos. Ejemplos: IP, RIP.
  • Capa de Enlace de Datos: Convierte los datos en binarios para su transmisión. Ejemplos: Ethernet, ARP.
  • Capa Física: Transmite los datos físicamente. Ejemplos: Bluetooth, USB.

    • Protocolos Más Utilizados

  • HTTP: Permite la transferencia de documentos en la web.
  • DNS: Convierte direcciones IP en nombres fáciles de recordar.
  • DHCP: Asigna direcciones IP automáticamente en una red.
  • SMTP: Facilita el envío y recepción de correos electrónicos.
  • ARP: Resuelve direcciones físicas (MAC) en una red local.
  • IP: Gestiona el envío de datos en paquetes, aunque no garantiza su recepción correcta.

    • Métodos de Transmisión de Datos

  • Unicast: Comunicación entre dos nodos específicos.
  • Multicast: Envío de datos a un grupo de nodos.
  • Broadcast: Transmisión simultánea a todos los nodos de la red.

    • Los protocolos son esenciales para garantizar la interoperabilidad y eficiencia en las redes modernas, permitiendo desde la navegación web hasta la transmisión de datos en tiempo real.
